## 🔍 Instrucciones de búsqueda (qué buscar y dónde)

| Si buscas... | Lo encuentras en... | Lo escribes como... |
|--------------|---------------------|----------------------|
| **Número de sensores** | Caso → "conformada por un ESP32, un sensor de temperatura DS18B20, un sensor de CO₂ Sensirion SCD30, un sensor de partículas PM2.5 Honeywell HPMA115S0" | `1x ESP32, 1x DS18B20, 1x SCD30, 1x HPMA115S0` |
| **Número de actuadores** | Caso → "display LCD 2004... dos LED: uno verde (statusLed) y uno rojo (alertLed)" | `1x LCD 2004, 1x LED verde, 1x LED rojo` |
| **Incertidumbre del sensor** | Datasheets → "Accuracy", "Tolerance", "±" | `±0.5°C` (DS18B20), `±(30ppm + 3%)` (SCD30 CO₂), `±15 µg/m³ o ±15%` (HPMA115S0) |
| **Exactitud/precisión de actuadores** | Caso + inferencia lógica (no hay datasheet de LEDs/LCD en el caso) | `Encendido en < 10 ms`, `mostrar valores completos sin truncamiento` |
| **Esfuerzo computacional** | Caso + inferencia por diseño | `ESP32: lectura de sensores, control de LEDs/LCD, envío HTTP al Edge Gateway` |

## ✅ Ejemplo llenado (caso AirWatch)

| Physical Layer Requirement | Especificación de requisitos |
|---------------------------|-------------------------------|
| **Número y tipos de nodos sensores y actuadores** | **Sensores:** 1x DS18B20 (temperatura), 1x SCD30 (CO₂, humedad, temperatura), 1x HPMA115S0 (PM2.5). <br> **Actuadores:** 1x LCD 2004, 1x LED verde (statusLed), 1x LED rojo (alertLed). <br> **Todos integrados en una AIAMU con ESP32.** |
| **Target uncertainty (incertidumbre objetivo)** | Según datasheets: <br> - DS18B20: **±0.5°C** en rango -10°C a +85°C <br> - SCD30 (CO₂): **±(30 ppm + 3% del valor)** en 400-10,000 ppm <br> - SCD30 (humedad): **±3% RH** <br> - HPMA115S0 (PM2.5): **±15 µg/m³** (0-100 µg/m³) o **±15%** (>100 µg/m³) |
| **Target accuracy and precision (exactitud y precisión objetivo) de los actuadores** | Según el caso: <br> - **LEDs:** Deben encender/apagar según `alertLedState` retornado por el Edge Gateway. La precisión es binaria (ON/OFF). <br> - **LCD 2004:** Debe mostrar valores de Temperatura (°C), CO₂ (ppm) y PM2.5 (µg/m³) completos y legibles. |
| **Processing Power (esfuerzo computacional)** | Según el caso: <br> - **Edge node (AIAMU con ESP32):** lectura periódica de los 3 sensores, control de LEDs y LCD, envío vía HTTP REST al Edge Gateway. <br> - **Fog node (Edge Gateway):** validación de umbrales por zona, retorno de `alertLedState`, almacenamiento temporal en caso de pérdida de conexión. |

## ⚡ Reglas rápidas (cheat sheet) para Pregunta 2

| Regla | Ejemplo |
|-------|---------|
| **Sensores:** copiar textual del caso | `1x DS18B20, 1x SCD30, 1x HPMA115S0` |
| **Actuadores:** copiar textual del caso | `1x LCD 2004, 1x LED verde, 1x LED rojo` |
| **Incertidumbre:** usar datasheet del sensor | `±0.5°C según datasheet DS18B20` |
| **Unidades correctas** | `°C`, `ppm`, `µg/m³`, `%RH` |
| **Actuadores:** si no hay datasheet, usar lógica de diseño | `mostrar valores sin truncamiento` |
| **Processing Power:** separar Edge vs Fog si aplica | `ESP32 (Edge) hace X; Edge Gateway (Fog) hace Y` |
